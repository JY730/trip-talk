유저 시나리오

페이지 진입 시 fetchBoard API를 통해 기존 게시글 데이터를 불러온다.

응답받은 zipcode, address, addressDetail을 각각의 인풋에 초기값으로 세팅한다.

사용자가 [우편번호 검색] 버튼을 클릭하면 react-daum-postcode를 활용해 모달을 띄운다.

새로운 우편번호를 선택하면 zipcode, address 값이 기존 값에서 변경된다.

[수정하기] 버튼 클릭 시 updateBoard API를 호출한다.

updateBoardInput에 zipcode, address, addressDetail을 포함한다.

주소 입력이 없더라도 정상적으로 수정 요청이 가능해야 한다.

수정 성공 시 “게시글이 성공적으로 수정되었습니다.” 메시지를 표시하고 상세페이지로 이동한다.

수정 실패 시 “게시글 수정 중 오류가 발생했습니다. 다시 시도해주세요.” 메시지를 표시한다.

export const useAddressEdit = (boardId: string) => {
// 1. 상태 관리: zipcode, address, addressDetail
// 2. fetchBoard 실행: 게시글 데이터 로드 후 상태 초기화
// 3. handleAddressSearch(): react-daum-postcode 모달 열기, 선택 결과로 zipcode, address 업데이트

// 5. handleUpdateBoard(): updateBoard API 호출, 성공 시 메시지 출력 + 상세페이지 이동, 실패 시 에러 메시지 출력
// 6. 반환 데이터 구조
            return {
                zipcode,
                address,
                addressDetail,
                handleAddressSearch,
                handleAddressChange,
                handleUpdateBoard
            }
            }
