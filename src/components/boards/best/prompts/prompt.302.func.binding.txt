핵심요구사항) 다음의 조건에 맞게 베스트 게시글 목록 조회 기능을 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.
            1) 라이브러리조건
                - API 통신: 이미 설치되어있는 apollo-client를 사용할 것.
                - 상태관리: React의 useState, useEffect 훅을 사용할 것.
                - 타입검증: GraphQL 타입을 기반으로 작성할 것.

            2) 유저시나리오
                1. 사용자가 /boards 페이지에 접속하면, GRAPHQL-API(ffetchBoardsOfTheBest)를 사용하여 게시글 목록 데이터를 조회할 것.
                    1-1) fetchBoardsOfTheBest API에서 반환된 실제 데이터를 화면에 렌더링할 것.
                    1-2) mock 데이터는 절대 사용하지 말 것.
                    1-3) 데이터 로드 중에는 “로딩 중입니다.” 문구를 화면에 표시할 것.
                    1-4) 데이터가 없을 경우 “베스트 게시글이 없습니다.” 문구를 표시할 것.

                2. 베스트 게시글 섹션에는 다음 정보가 모두 포함되어야 함.
                    - 대표 이미지 (API 응답에 포함된 images[0])
                    - 게시글 제목
                    - 작성자(writer)
                    - 좋아요 수(likeCount)
                    - 작성일(createdAt)
                
                3. 각 게시글 카드는 클릭 가능한 영역으로 구성할 것.
                    - 클릭 시 router.push(`/boards/${id}`)를 통해 상세페이지로 이동할 것.
                    - 라우팅 동작이 실제로 작동하는지 테스트에서 검증할 것.
                
                4. 네트워크 에러 또는 통신 실패 시
                    - “베스트 게시글을 불러오는 중 오류가 발생했습니다. 다시 시도해주세요.” 메시지를 화면에 표시할 것.
                    - try-catch문을 활용할 것.

핵심요구사항) 테스트 구현 조건
            1) playwright 기반으로 테스트를 작성할 것.
            2) 테스트 파일 경로: src/components/boards/tests/index.binding.hook.spec.ts
            3) 테스트 시나리오

                - fetchBoardsOfTheBest 요청이 정상적으로 수행되어 베스트 게시글이 표시되는지 검증할 것.
                - 게시글 제목 클릭 시, 해당 게시글 상세페이지로 이동하는지 검증할 것.
                - API 호출 실패 시 오류 메시지가 화면에 표시되는지 검증할 것.
            4) timeout 설정

                - UI 렌더링 테스트: 500ms 미만
